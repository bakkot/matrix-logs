<Sam>

Proposal for disabling of automatic updates. I've rewritten the requirement as well as the assessment:

5.5.1 REQ-SU-1  The product shall provide an easy-to-use option for the user to disable or defer automatic updates.

Note: For UC-INST this user can be the enterprise administrator  Applicability: Browser does not rely on OS for secure update  # Assessment ASS-SU-1

Reference: 

Assessment of [REQ-SU-1] 
Objective

Assess that users have an easy-to-use option for disabling or deferring automatic updates of the browser.

Preparation

- Browser in a state capable of being updated (eg build/configuration pointing at test update server, or an old build in need of update)
- Test process for triggering automatic browser update
- Locate the relevant configuration interface based on technical documentation for the product

Activities

1. Navigate to interface element
2. Disable automatic updates
3. Trigger automatic update process
4. Evaluate whether an automatic update occurs
5. Navigate to interface element and enable automatic updates
6. Trigger automatic update process
7. Evaluate whether an automatic update occurs

Evaluation

Pass:
    - Configuration element is in intuitive location (eg on a main configuration screen, or part of update process UI)
    - Browser does not perform an automatic update when disabled
    - Browser performs automatic update when enabled
Fail: Any of the above are not fulfilled.
