here is the updated assessment for ACC-STORE-LOG-1:

6.14.3.2 Objective Assess that the product provides an interface for viewing information about stored data at a granularity of site or narrower.

6.14.3.3 Preparation

  • Prepare tooling to inspect the data stored within the browser.
  • User documentation describing the process for viewing stored data.
  • Website on site A that will store data, including:
    • cookie data
    • LocalStorage
    • IndexedDB storage
  • Website on site B that will store data, including:
    • cookie data
    • LocalStorage
    • IndexedDB storage

6.14.3.4 Activities

  1. Visit website on site A, and verify via tooling it has set data.
  2. Visit website on site B, and verify via tooling it has set data.
  3. Follow user documentation to open the product interface for viewing stored data.
  4. Verify the interface displays information about data stored for site A.
  5. Verify the interface displays information about data stored for site B.
  6. Verify the information is attributed per site or narrower (origin), not aggregated at a coarser granularity.

6.14.3.5 Verdict

  • Pass:
    • Data is set successfully during website visits, confirmed via tooling.
    • The product provides an interface for viewing information about stored data.
    • The interface presents information for both site A and site B.
    • Information is presented at a granularity of site or narrower (origin).
  • Fail: Any of the above are not fulfilled.

6.14.3.6 Evidence

  • Screenshot or log output from tooling showing ground-truth stored data.
  • Screenshot of the product interface showing per-site or per-origin information.
  • User documentation providing instructions for viewing stored data.
