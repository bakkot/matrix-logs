6.5.4 ACC-EXT-SU-2 

6.5.4.1 Reference 

Assessment of \[REQ-EXT-SU-2\] 

The product shall enable users to decline automatic updates of any or all extensions. 

6.5.4.2 Objective 

Assess that the user can decline automatic updates of an extension, and that a declined extension is not automatically updated. 

6.5.4.3 Preparation 

Reset browser to factory default settings. 

A test extension prepared in two versions, a lower version and a higher version. 

An extension distribution channel to which the test extension can be uploaded. 

 

6.5.4.4 Activities 

The following steps are to be carried out in order: 

Upload the lower version of the test extension to the extension distribution channel. 

Install the test extension into the product from the distribution channel. 

Record the installed version shown on the product's extension configuration page. 

Locate the setting to decline automatic updates, and use it to decline automatic updates of the extension. 

Upload the higher version of the test extension to the extension distribution channel. 

Trigger or await an automatic update cycle. 

Verify the installed version shown on the product's extension configuration page. 

 

6.5.4.5 Verdict 

Pass: The product provides a setting to decline automatic updates, and the extension remains at the lower version after the higher version is published. 

Fail: No such setting is provided, or the extension is automatically updated to the higher version. 

 

6.5.4.6 Evidence 

Screenshot(s) of the setting used to decline automatic updates. 

Screenshot(s) of the product's extension configuration page showing the installed version before and after the higher version is published.
